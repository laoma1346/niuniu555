
## 第1步：C++编译（前置条件）

### 1.1 编译项目

**【操作】**
```
Step 1: 关闭 UE5 编辑器（必须！）
Step 2: 打开 Visual Studio
Step 3: 按 Ctrl+Shift+B 编译
Step 4: 等待显示 ====== Build: 1 succeeded ======
```

**【验证】**
- 输出窗口显示: `Result: Succeeded`
- 无错误无警告

### 1.2 重启编辑器

**【操作】**
```
Step 5: 重新打开 UE5 编辑器
Step 6: 确认项目正常加载
```

**【故障排除】**
| 问题 | 解决 |
|------|------|
| 编译失败 | 查看错误列表，检查代码语法 |
| 找不到UnrealEditor-xxx.dll | 关闭UE5后再编译 |

## 第2步：创建数据资产（Data Asset）

### 2.1 创建敌人数据资产

**【操作路径】** 内容浏览器右键 → `Miscellaneous` → `Data Asset`

**【步骤】**
```
Step 1: 右键内容浏览器空白处
Step 2: 选择 Miscellaneous（杂项）
Step 3: 选择 Data Asset（数据资产）
Step 4: 搜索并选择 "EnemyDataAsset"
Step 5: 命名为 "DA_TestEnemy"
Step 6: 【验证】图标为白色文档 📄（不是蓝色方块！）
```

**【故障排除】**
| 问题 | 原因 | 解决 |
|------|------|------|
| 搜不到EnemyDataAsset | 未编译或缓存问题 | 重新编译+重启编辑器 |
| 图标是蓝色方块 | 创建成了蓝图类 | 删除，重新按Miscellaneous→Data Asset创建 |

### 2.2 配置基础属性（必填）

**【操作】** 双击 `DA_TestEnemy` 打开配置面板

**【配置顺序】**
```
1. 敌人基础（从上到下）
   ├── 敌人ID: TestEnemy_001
   ├── 敌人名称: 测试小怪
   ├── 敌人类型: Normal
   └── 基础等级: 1

2. 基础属性（核心！）
   ├── 基础生命值: 100          【必须>0】
   ├── 基础攻击力: 10
   ├── 基础防御力: 5
   └── 基础移动速度: 300

3. 视觉资源（必须配置，否则看不到模型）
   ├── 骨骼网格体: [拖拽 SK_Mannequin]
   └── 动画蓝图: [拖拽 ABP_Mannequin]

4. 受击配置
   ├── 受击硬直时间: 0.3
   ├── 可被击退: ☑️
   └── 可被眩晕: ☑️

5. 死亡配置
   ├── 尸体存在时间: 5
   └── 启用布娃娃: ☑️
```

**【验证】**
- 所有数值已填写（无空白）
- 骨骼网格体已指定（无None）
- 按 Ctrl+S 保存

---

## 第3步：创建敌人蓝图

### 3.1 创建蓝图类

**【操作路径】** 内容浏览器右键 → `Blueprint Class`

**【步骤】**
```
Step 1: 右键内容浏览器
Step 2: 选择 Blueprint Class（蓝图类）
Step 3: 展开 "所有类"（All Classes）
Step 4: 搜索 "EnemyBase"
Step 5: 选择 AEnemyBase
Step 6: 命名为 "BP_TestEnemy"
Step 7: 【验证】图标为蓝色方块 🔷
Step 8: 双击打开蓝图编辑器
```

### 3.2 检查继承组件（关键！）

**【操作】** 查看左上角 `组件` 面板

**【预期结果】**
```
组件面板应显示（灰色图标=继承，不可删除）:
├── Mesh                          【灰色】
├── CharacterMovementComponent    【灰色】
├── AttributeComp                 【灰色】✅ 属性组件
├── HitReactionComp               【灰色】✅ 受击反馈组件
└── StateMachineComp              【灰色】✅ 状态机组件
```

**【故障排除】**
| 问题 | 原因 | 解决 |
|------|------|------|
| 缺少某个组件 | 未从EnemyBase继承 | 删除重建，确保继承EnemyBase |
| 有两个同名组件 | 重复添加了 | 删除白色图标的（手动添加的） |
| 组件是白色图标 | 误手动添加 | 右键删除，保留灰色图标的 |

### 3.3 配置 Mesh 组件

**【操作】** 组件面板 → 点击 `Mesh` → 细节面板

**【配置】**
```
骨骼网格体: [选择 SK_Mannequin]     【必须配置】
动画类: [选择 ABP_Mannequin]         【必须配置】
```

### 3.4 配置 HitReactionComp 组件

**【操作】** 组件面板 → 点击 `HitReactionComp` → 细节面板

**【配置】** 按以下数值填写：
```
受击反馈|闪白:
├── Flash Duration: 0.15          【闪白持续时间】
└── Flash Color: (1, 0.3, 0.3)    【红色】

受击反馈|顿帧:
├── Hit Stop Duration: 0.05       【顿帧时间】
└── Hit Stop Time Dilation: 0.05  【慢动作效果】

受击反馈|击退:
├── Knockback Speed: 800          【击退速度】
└── Knockback Deceleration: 2000  【击退减速】
```

### 3.5 关联数据资产

**【操作路径】** 蓝图编辑器 → `类默认值`（右上角）

**【步骤】**
```
Step 1: 点击 "类默认值" 按钮
Step 2: 左侧面板搜索栏输入 "enemy"
Step 3: 找到 "Enemy Data" 字段
Step 4: 下拉选择 "DA_TestEnemy"
Step 5: 【验证】显示 DA_TestEnemy（已设置）
```

### 3.6 启用调试显示

**【操作】** 仍在 `类默认值` 面板

**【步骤】**
```
Step 6: 搜索栏改为输入 "show"
Step 7: 找到 "Show Debug Info"
Step 8: ☑️ 勾选 Show Debug Info         【必须勾选！】
Step 9: （可选）勾选 Show Debug Shapes
```

**【验证】**
```
类默认值面板应显示:
├── Enemy Data: DA_TestEnemy
├── Show Debug Info: ☑️
└── 当前章节: 1
```

### 3.7 编译保存

**【操作】**
```
Step 10: 点击工具栏 "编译"（Compile）按钮
Step 11: 等待绿色对勾 ✅
Step 12: 点击 "保存"
Step 13: 关闭蓝图编辑器
```

**【故障排除】**
| 问题 | 原因 | 解决 |
|------|------|------|
| 编译失败（红色×） | 配置错误 | 查看输出日志，检查必填项 |
| 找不到Enemy Data | 未保存数据资产 | 保存DA_TestEnemy后再选 |

## 第5步：运行验证（按优先级）

### 5.1 打开日志窗口

**【操作路径】** 窗口 → 开发者工具 → `输出日志`

**【步骤】**
```
Step 1: 打开输出日志窗口
Step 2: 在日志窗口的搜索栏输入 "EnemyBase"（筛选日志）
Step 3: 运行游戏（Alt+P 或点击Play）
```

### 5.2 Phase 1: 血量初始化验证（必须首先通过！）

**【检查项】**

| 检查点 | 预期结果 | 状态 |
|--------|----------|------|
| 日志输出 | `[EnemyBase] XXX BeginPlay 开始初始化` | ⬜ |
| 初始化完成 | `[EnemyBase] XXX 初始化血量: 100/100` | ⬜ |
| 头顶显示 | `[BP_TestEnemy] HP:100/100 \| State:Spawn` | ⬜ |
| 状态流转 | 自动变为 `State:Idle` | ⬜ |

**【失败排查】**
```
现象: 显示 HP:0/0
原因1: DA_TestEnemy未配置生命值
解决: 双击DA_TestEnemy → 基础生命值设为100 → 保存

原因2: Enemy Data未关联
解决: 打开BP_TestEnemy → 类默认值 → Enemy Data选择DA_TestEnemy

原因3: 未重新编译C++
解决: 关闭UE5 → VS编译 → 重启UE5
```

### 5.3 Phase 2: 受击扣血验证

**【操作】** 用武器攻击敌人

**【检查项】**

| 检查点 | 预期日志 | 状态 |
|--------|----------|------|
| TakeDamage调用 | `[EnemyBase] TakeDamage被调用, 原始伤害:171` | ⬜ |
| 伤害计算 | `[EnemyBase] 计算后实际伤害:XX` | ⬜ |
| 血量变化 | `[EnemyBase] 血量变化: 100 -> XX` | ⬜ |
| 血量事件 | `[EnemyBase] 血量变化: XX/100 (变化: -XX)` | ⬜ |
| 头顶显示 | HP值减少（如HP:70/100） | ⬜ |

**【失败排查】**
```
现象: 攻击无反应（无日志）
原因: 武器碰撞通道不匹配
解决: 检查武器和敌人的碰撞设置

现象: 血量不变（日志显示变化但头顶不变）
原因: GetCurrentHealth未正确实现
解决: 确认C++代码已编译并重启
```

### 5.4 Phase 3: 状态机切换验证

**【操作】** 攻击命中后继续观察

**【检查项】**

| 检查点 | 预期日志 | 状态 |
|--------|----------|------|
| 进入受击 | `[EnemyBase] 进入受击状态` | ⬜ |
| 状态变更 | `[EnemyStateMachine] Idle -> Hit` | ⬜ |
| 头顶显示 | `State:Hit` | ⬜ |
| 硬直结束 | 自动恢复为 `State:Idle` | ⬜ |
| 状态恢复 | `[EnemyStateMachine] Hit -> Idle` | ⬜ |

**【失败排查】**
```
现象: 不进入Hit状态
原因: HitInfo.HitReaction为None
解决: 武器攻击时设置正确的HitReaction类型

现象: 卡在Hit状态不恢复
原因: 硬直时间配置过长
解决: DA_TestEnemy → 受击硬直时间设为0.3
```

### 5.5 Phase 4: 受击反馈验证

**【检查项】**

| 检查点 | 预期结果 | 状态 |
|--------|----------|------|
| 触发反馈 | `[EnemyBase] 触发受击反馈` | ⬜ |
| 闪白效果 | 敌人模型瞬间变红/白（0.15秒） | ⬜ |
| 击退效果 | 敌人位置后退 | ⬜ |
| 音效播放 | 听到受击音效（如配置了） | ⬜ |
| 顿帧效果 | 暴击时游戏时间变慢 | ⬜ |

**【失败排查】**
```
现象: 无闪白
检查: HitReactionComp → Flash Duration是否为0
解决: 设为0.15

现象: 无击退
检查: 武器HitInfo.KnockbackForce是否>0
解决: 武器数据资产设置击退力度

现象: 无音效
检查: DA_TestEnemy是否配置了HitSound
解决: 拖拽音效资源到HitSound字段
```

### 5.6 Phase 5: 死亡验证

**【操作】** 连续攻击直到血量归零

**【检查项】**

| 检查点 | 预期日志 | 状态 |
|--------|----------|------|
| 血量归零 | `[EnemyBase] 血量归零，触发死亡` | ⬜ |
| 死亡状态 | `[EnemyStateMachine] Hit -> Dead` | ⬜ |
| 头顶显示 | `State:Dead` | ⬜ |
| 死亡动画 | 播放死亡动画/布娃娃 | ⬜ |
| 尸体消失 | 5秒后敌人消失 | ⬜ |

**【失败排查】**
```
现象: 血量到0但不死亡
原因: bIsDying标志未正确设置
解决: 检查代码逻辑

现象: 无死亡动画
原因: DA_TestEnemy未配置DeathMontage
解决: 配置死亡动画蒙太奇
```

## 附录：快速诊断表

| 现象 | 检查点 | 解决方案 |
|------|--------|----------|
| HP:0/0 | DA_TestEnemy生命值配置 | 设为100 |

| 无状态切换 | HitInfo.HitReaction | 设置为Light/Heavy |
